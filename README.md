# QuickIsland-M1-Pedidos
Estructura del modulo numero1 del proyecto QuickIsland-Cofee , Gestion de pedidos

El sistema QuickIsland-Coffee es una plataforma integral diseñada para optimizar la gestión de pedidos a demanda dentro de un entorno académico. El proyecto implementa un modelo operativo basado en islas de especialidad, donde cada punto de venta gestiona un catálogo diferenciado de productos para la comunidad estudiantil.
Objetivo y Problema que Resuelve

    Optimización del Tiempo Estudiantil: El sistema aborda la problemática de los horarios académicos inflexibles, permitiendo que los alumnos con periodos de descanso limitados realicen pedidos de forma remota.

    Eficiencia Operativa: Reduce las aglomeraciones en los puntos de venta físicos y elimina los tiempos de espera prolongados mediante un flujo de recolección ágil.

    Control Administrativo: Proporciona a los gestores de cada isla herramientas avanzadas para la administración de inventarios en tiempo real y la supervisión de pedidos entrantes.

Experiencia del Alumno (Frontend)

El módulo permite a los usuarios interactuar con una interfaz dinámica donde pueden seleccionar la isla de su preferencia, explorar el catálogo actualizado y gestionar un carrito de compras personalizado. Esta autonomía permite al estudiante planificar su consumo conforme a su disponibilidad horaria, recibiendo notificaciones sobre el estado de su pedido.
Gestión de la Isla (Backend y Administración)

Para los administradores, el sistema ofrece un panel de control robusto con las siguientes capacidades:

    Gobernanza de Inventario: Capacidad para dar de alta nuevos productos, modificar precios y descripciones, o retirar artículos del catálogo según la disponibilidad física.

    Monitoreo de Pedidos Online: Visualización centralizada de las solicitudes entrantes, permitiendo una preparación anticipada que agiliza la entrega final.

    Escalabilidad Comercial: La automatización de procesos abre la posibilidad de ampliar la oferta de productos y atraer a un mayor volumen de clientes gracias a la eficiencia en el servicio.

Seguridad e Integridad de Datos

El proyecto destaca técnicamente por la implementación de Triggers de Auditoría y Transacciones SQL, garantizando que cada pedido online sea trazable y que el inventario se actualice con precisión matemática, evitando discrepancias en los registros financieros y de almacén.
